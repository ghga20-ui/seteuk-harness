# seteuk-harness

수행평가 학생 원본으로 과목별 세부능력 및 특기사항(과세특) 초안을 생성하는
Claude Code 스킬입니다. 쓰면 쓸수록 사용자의 문체와 활동에 맞춰 정교해집니다.

## 사용 철학 — 이 도구는 평가를 대신하지 않습니다

세특 작성의 주체는 평가자인 교사입니다. 이 하네스는 **평가를 마친 교사가 그 결과를
기록으로 옮기는 일**을 돕는 도구이며, 그 경계를 동작으로 강제합니다.

- **채점 없이는 생성 없음**: 교사의 평가 자료(채점표, 루브릭 점수, 관찰 기록)가
  확인되지 않은 활동은 세특 생성을 거부합니다(검증기가 저장 단계에서도 차단)
- **수준 판정의 주권은 교사에게**: 세특의 톤(수준)은 교사의 평가에서 파생되며,
  AI의 조정 의견은 반드시 생성 전에 질문으로 제시되고 승인된 것만 반영됩니다
- **모든 산출물은 "AI 초안"으로 표시**되며, 교사가 검수를 완료하기 전에는
  확정본이 만들어지지 않습니다. 검수 전 NEIS 입력은 금지입니다
- 교사의 수업 중 관찰 메모를 입력하면 세특에 결합됩니다 — 학생 산출물에 없는
  교사만의 관찰이야말로 세특을 기록답게 만드는 재료입니다

이 설계는 Human in the Loop 원칙을 따릅니다. 여러분(교사)이 결정하는 지점은
채점, 규칙·구조 인터뷰, **파일럿 배치 승인**(표본 3~4명을 먼저 보고 승인해야
나머지가 생성됨), 검수(우선 검토 대상부터), 회고·확정 선언입니다.
반복 작업(배치 생성, 규칙 검증)만 기계가 맡습니다. 사람을 "승인 버튼 누르는
기계"로 만드는 학생별 개별 승인 같은 설계는 의도적으로 배제했습니다.

## 특징
- 규칙 인터뷰 + 구조 샘플 시연: 학생 원본을 분석해 세특 구조 후보를 실제 완성 문장으로 비교 선택
- 2단 검증 게이트: 바이트·금지어·문두는 스크립트가, 근거 충실성·개별성·톤 일치는 LLM 루브릭이 검사.
  검증을 통과해야만 xlsx가 저장됩니다
- 성장하는 위키: 회차마다 인터뷰 결정·채택 구조·검수 수정 패턴이 wiki에 쌓여
  다음 회차의 인터뷰가 짧아지고 표현이 사용자 문체에 수렴합니다

## 설치 (제일 쉬운 방법 — 복사해서 붙여넣기)
Claude Code를 열고 아래 한 줄을 붙여넣으세요. 설치는 Claude가 대신 해줍니다.

```
https://github.com/ghga20-ui/seteuk-harness 스킬을 내 스킬 폴더(~/.claude/skills/seteuk-harness)에
설치해줘. 끝나면 사용법을 알려줘.
```

파이썬 등 필요한 프로그램은 스킬이 첫 실행 때 스스로 확인하고 설치합니다.
설치가 끝나면 "세특 작성해줘" 또는 "과세특 생성"이라고 요청하는 순간 스킬이 동작합니다.

<details>
<summary>직접 설치하고 싶다면</summary>

1. 이 저장소를 `C:\Users\<사용자>\.claude\skills\seteuk-harness`(또는 `~/.claude/skills/seteuk-harness`)에 클론하거나 zip으로 받아 풉니다
2. Python 3.10+ 와 openpyxl: `python -m pip install openpyxl` (생략해도 스킬이 첫 실행 때 자동 설치)
</details>

> Codex 등 다른 코딩 에이전트 사용자: 자동 트리거는 Claude Code 전용이지만,
> 내용물이 전부 마크다운과 파이썬이라 "이 폴더의 SKILL.md를 읽고 절차를 따라줘"라고
> 수동으로 지시하면 사용할 수 있습니다(공식 지원 범위는 Claude Code입니다).

## 지원하는 학생 원본 포맷
별도 MCP나 OCR 도구 없이 다음을 처리합니다.
- xlsx, hwpx, 확장자만 다른 HTML 파일, 일반 텍스트: 번들 추출기(`scripts/extract_sources.py`)가 처리
- 사진(jpg, png), 스캔 pdf: Claude가 파일을 직접 읽어 손글씨까지 전사
- hwp 바이너리: 유일한 예외 — 한글 프로그램에서 hwpx나 pdf로 다시 저장해 주세요
- 한글 문서 파싱 MCP(kordoc 등)가 설치되어 있으면 자동으로 활용하지만, 없어도 됩니다

## 사용 흐름
학생 원본과 채점표 위치 제시 → 규칙 인터뷰(7문항 이내) → 구조 시연에서 선택 →
생성·검증 → xlsx 초안 수령 → 검수 후 수정본을 주면 회고가 위키에 학습 → 확정본 발행

한 번 설치하면 스킬 폴더는 여러분 것입니다. 세특을 쓸수록 폴더 안의 wiki에
여러분의 문체와 결정이 쌓여 다음 작업이 점점 짧고 정확해집니다.
새 버전으로 교체하고 싶을 때는 기존 `wiki` 폴더를 보관해 두었다가
교체 후 되돌려 넣으면 성장분이 유지됩니다.

## 반드시 지켜야 할 것
- **초안을 그대로 NEIS에 입력하지 마세요.** 이 스킬의 산출물은 어디까지나 초안이며,
  실제 수행과의 일치, 허위·과장 여부, 기재요령 준수의 최종 확인 책임은 교사에게 있습니다
- 학생 명렬을 제공하면 학번 오매핑(다른 학생에게 세특이 귀속되는 사고)과
  미제출자 누락을 검증기가 자동으로 잡아줍니다 — 명렬 제공을 권장합니다

## 데이터 경계
- 스킬 위키에는 학생 실명·학번·원문이 기록되지 않습니다(익명 통계만)
- 학생 데이터가 담긴 산출물(xlsx, 활동프로파일.json 등)은 전부 여러분의 작업 폴더에 남습니다
- 사용하면서 쌓이는 위키(활동이력, 문체관찰 등)는 여러분 컴퓨터의 스킬 폴더 안에만 존재합니다.
  스킬 폴더를 다른 사람에게 통째로 전달할 일이 있다면 `wiki/활동이력/`의 기록과
  `wiki/문체관찰.md`의 내용을 비우고 전달하세요(여러분의 검수 습관이 담긴 개인 데이터입니다)

## 문제 해결
- 파이썬·openpyxl은 스킬이 첫 실행 때 자동으로 확인·설치합니다. 자동 설치가 실패하면
  python.org 또는 Microsoft Store에서 Python 3.10+ 설치 후 `python -m pip install openpyxl`
- pip 설치가 막히는 환경(학교 프록시 등): 관리자에게 문의하거나 개인 네트워크에서 설치
- 학급이 많은 경우(3개 학급 이상): 반 단위로 나눠 요청하세요
